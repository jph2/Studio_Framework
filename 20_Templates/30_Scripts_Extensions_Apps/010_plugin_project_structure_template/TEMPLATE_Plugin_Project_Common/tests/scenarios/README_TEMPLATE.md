# ⚠️ TEMPLATE FILE — DO NOT SHIP AS-IS

Scenario tests define behavioral outcomes independent of implementation details.

## Scenario format

Each scenario file should include:

- `id`: stable scenario id
- `objective`: what behavior is validated
- `inputs`: input scene/data/config
- `expected`: measurable outputs (files, logs, structure, status)

## Minimum requirement

- At least one scenario file for the primary workflow.
