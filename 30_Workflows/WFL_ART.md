## Purpose

Defines the core artifact maturation workflow.

## Pipeline
1. Discovery + Research (shared base phase)
2. Branch into one or more prep directions as needed:
   - Tutorial Prep
   - Paper Prep
   - Project Prep
3. In Project Prep, use Requirements as the default structured continuation artifact
4. Add Specification only when deeper design detail is genuinely needed
5. Create an Implementation Plan when the work is ready for executable sequencing
6. Build / incubate / review
7. Graduate to a standalone repo or implementation space when the work has enough real project gravity

## Rule
Do not jump directly from vague idea to coding when the problem still needs framing.
